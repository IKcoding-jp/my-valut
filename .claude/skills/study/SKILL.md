# 学習セッション管理

## 起動時の確認

ユーザーに「開始ですか？終了ですか？」と確認する。
文脈から判断できる場合は確認不要。

## 学習開始フロー

1. **試験カウントダウン表示**
   - 基本情報技術者試験までの残り日数

2. **今日の学習範囲を提案**
   - `learning/基本情報技術者/学習計画表.md` を読み、計画に基づく今日の範囲を提示
   - `learning/基本情報技術者/進捗ログ.md` を読み、前回の続きから提案
   - 試験14日前以内: 過去問演習を中心に提案
   - 試験7日前以内: 弱点分野の集中復習を提案

3. **簡潔に表示**
   ```
   | 試験まで | X 日 |
   | 今日の範囲 | ... |
   | 前回の続き | ... |
   ```

## 学習終了フロー

1. **学習内容をヒアリング**
   - 何を勉強したか（範囲・テーマ）
   - Anki復習した場合は枚数
   - 学習時間（わかれば）
   - 理解度・手ごたえ（任意）

2. **進捗ログに記録**
   - `learning/基本情報技術者/進捗ログ.md` に追記
   - 日付、学習内容、時間を記録

3. **デイリージャーナルに反映**
   - 当日のジャーナル（`journal/YYYY/MM/YYYY-MM-DD.md`）が存在すれば、学習記録セクションを更新

4. **フィードバック**
   - 学習の継続を具体的に認める（「X日連続で学習できていますね」など）
   - 次回の学習予定を簡潔に伝える

## FE試験の詳細スキル（基本情報技術者試験）

FE学習セッション時は、以下のリファレンスを必要に応じて参照する：

- `references/fe-study.md` - FE学習セッションの詳細フロー（問題出題・解説・メタ認知）
- `references/dashboard.md` - ダッシュボード更新手順・優先度計算
- `references/metacognition.md` - メタ認知記録・エラーパターン分類
- `references/mock-exam.md` - 模試モード定義・弱点分析
- `references/rubric.md` - 解答評価ルーブリック

FE学習の追加ファイル：
- `memory_logs.md` - セッション間の長期記憶
- `learning/基本情報技術者/学習記録/` - 日次の詳細ログ
- `learning/基本情報技術者/questions/` - 過去問ファイル（YAML frontmatter管理）

## CSV受信時の自動処理（最重要）

**ユーザーがCSVファイルパスを共有した瞬間**に、以下を確認なしで自動実行する：

1. **CSVをcp932でデコード**して正誤・分野を集計する
2. **学習記録**（`learning/基本情報技術者/学習記録/YYYY-MM-DD.md`）に結果を追記する
   - ファイルがなければ新規作成
3. **進捗ログ**（`learning/基本情報技術者/進捗ログ.md`）を更新する
   - 分野別詳細の正答数・問題数・正答率
   - 系統別サマリーの再計算
   - 学習セッション履歴に行を追加
   - frontmatter（updated, total_questions_attempted, total_correct, overall_accuracy, days_remaining）
4. **学習計画表**（`learning/基本情報技術者/学習計画表.md`）の該当日セルを実績で上書きする
5. 完了後「記録しました」と**一言だけ報告**して、次の行動を提案する

**絶対に「記録しますか？」と聞かない。** 黙って記録して、結果だけ伝える。

## セッション終了時の自動処理

「お疲れ」「終わり」「いってきます」等の終了シグナルで自動実行：

1. 当日の学習記録を最終化する（振り返り・心理状態を追記）
2. **memory_logs.md** にその日の要点を追記する
3. git commit & push（daily-reportスキルと連携）

## 注意事項

- 学習時間が短くても否定しない。「やったこと自体」を認める
- 基本情報技術者とコーヒーマイスターの両方に対応する
- 進捗ログのフォーマットは既存の記述スタイルに合わせる
