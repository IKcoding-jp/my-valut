# 解答評価ルーブリック

## 評価レベル表

| レベル | 判定 | 基準 |
|--------|------|------|
| 1 | 不合格 | 選択肢を選んだだけ、または「なんとなく」 |
| 2 | 不合格 | キーワードは言えるが論理構造がない |
| 3 | 合格 | 正解理由を因果関係で説明できる |
| 4 | 合格 | 不正解選択肢の排除理由も説明できる |
| 5 | 優秀 | 関連概念との接続、応用例まで言及 |

## タイプ別合格基準

| 科目 | タイプ | 最低合格レベル |
|------|--------|----------------|
| 科目A | 暗記系 | レベル3 |
| 科目A | 理解系 | レベル4 |
| 科目A | 混合系 | レベル3 |
| 科目B | アルゴリズム | レベル4 + トレース説明 |
| 科目B | セキュリティ | レベル4 |

## レベル判定の具体例

### レベル1の例
- 「イだと思います」（根拠なし）
- 「なんとなくウっぽい」

### レベル2の例
- 「暗号化だからウです」（キーワードだけ）
- 「SQLインジェクションは聞いたことあるのでア」

### レベル3の例
- 「共通鍵暗号は送信者と受信者が同じ鍵を使うので、鍵の受け渡しが課題になる。だからウが正解」
- 「スタックはLIFO（後入れ先出し）なので、最後に入れたデータが最初に出る。だからイ」

### レベル4の例
- 「ウが正解。アはハッシュ関数の説明で暗号化ではない。イは公開鍵暗号の特徴。エはデジタル署名の説明」
- 「線形探索は先頭から順に探すのでO(n)。二分探索はソート済みデータを半分ずつ絞るのでO(log n)。だからイ。アはO(1)でハッシュ探索、ウとエは存在しない計算量」

### レベル5の例
- 「共通鍵暗号のAESは処理が速く大量データ向き。公開鍵暗号のRSAは鍵配送問題を解決するが遅い。実際のHTTPS通信ではハイブリッド方式（RSAで共通鍵を交換→AESで通信）を使う」

## 「習得済」判定基準

問題の status を「習得済」に変更する条件:
- 正解 **かつ** 評価レベル3以上
- 2回連続で上記を満たした場合は確実に「習得済」
