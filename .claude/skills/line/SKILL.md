# LINEメモ整理

## ワークフロー

1. **LINE/ディレクトリをスキャン**
   - `LINE/` 内のすべての `.md` ファイルを一覧取得
   - 各ファイルの内容を読み取る

2. **内容を分類**
   - 各メモの内容を分析し、以下の振り分け先を提案:

   | 内容の種類 | 振り分け先 |
   |-----------|-----------|
   | アイデア・思いつき | `ideas/` に追記または新規作成 |
   | タスク・やること | `tasks/タスク.md` に追記 |
   | 学習メモ | `learning/` の該当ファイルに追記 |
   | 感想・レビュー | `entertainment/` または `reading/` に追記 |
   | 個人的なメモ | `journal/` の該当日に追記、または `ideas/` |
   | 分類不明 | ユーザーに確認 |

3. **ユーザーに提案を表示**
   - 各メモの概要と提案先を表形式で一覧表示:
   ```
   | ファイル | 内容の概要 | 提案先 |
   |---------|-----------|--------|
   | 2026-01-15-xxx.md | アプリのアイデア | ideas/やりたいこと.md |
   | ... | ... | ... |
   ```

4. **確認後に実行**
   - ユーザーの承認を得てから移動・統合を実行
   - 既存ファイルへの追記の場合、適切なセクションに挿入
   - 新規ファイルの場合、命名規則に従って作成

## 注意事項

- `LINE/` 内のファイルは**直接編集しない**（自動同期のため）
- 内容を他のファイルにコピー・統合する形で整理する
- 統合後、元のLINEファイルは残す（同期が壊れるため削除しない）
- 1つのメモに複数の内容が含まれる場合、それぞれ適切な場所に振り分ける
