# クイック解説（quick-explain）

学習セッション外で、教科書を読んでいて分からない用語・概念が出てきたときにサッと使える解説スキル。
フルの学習セッションを起動する必要がなく、1回の応答で完結する。

---

## 解説フォーマット

### 単一用語の解説

```
📖 [用語名]
━━━━━━━━━━━━━━━━━━━━━

🔰 ひとことで言うと:
  [1行で超簡潔に]

🏠 身近な例え:
  [日常生活の例えで説明]

📝 試験ではこう出る:
  [試験での問われ方と覚え方のコツ]

🎯 これだけ覚えればOK:
  ・[ポイント1]
  ・[ポイント2]

━━━━━━━━━━━━━━━━━━━━━
```

### 2つの概念の比較

```
📖 [概念A] vs [概念B]
━━━━━━━━━━━━━━━━━━━━━

🔰 ひとことで言うと:
  [A] = [1行で]
  [B] = [1行で]

🏠 身近な例え:
  [A]は○○、[B]は△△みたいなもの

📝 見分け方（試験のコツ）:
  ・「□□」というキーワードが出たら → [A]
  ・「■■」というキーワードが出たら → [B]

| | [A] | [B] |
|---|---|---|
| 目的 | ... | ... |
| 特徴 | ... | ... |
| 覚え方 | ... | ... |

━━━━━━━━━━━━━━━━━━━━━
```

### 計算・手順系の解説

```
📖 [テーマ名]
━━━━━━━━━━━━━━━━━━━━━

🔰 ひとことで言うと:
  [何をする計算/手順なのか]

👆 指折りで解く方法:
  ステップ1: [やること]
  ステップ2: [やること]
  ステップ3: [やること]
  → 答え: [結果]

⚡ 試験の裏技:
  [計算しないで解く方法があれば提案]

━━━━━━━━━━━━━━━━━━━━━
```

---

## 注意事項

- CLAUDE.mdの「回答のルール（FE学習時）」を厳守する

- セッション開始手順は不要（軽量スキル）
- ファイル更新は行わない
- 1回の応答で完結する（追加の対話は最小限）
- 出力はスクロールせずに読める量に収める
- 説明が長くなりそうなら「まずはこれだけ」に絞り、「もっと知りたい？」と聞く
- もっと深く学びたい場合は「○○を勉強したい」と言えば deep-learn スキルが起動する旨を案内する
