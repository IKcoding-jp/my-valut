# クイック解説（quick-explain）

学習セッション外で、教科書を読んでいて分からない用語・概念が出てきたときにサッと使える解説スキル。
フルの学習セッションを起動する必要がなく、1回の応答で完結する。

---

## トリガー

以下のような発言で起動する:
- 「○○って何？」「○○とは？」
- 「○○を教えて」「○○がわからない」
- 「○○と△△の違いは？」
- 「○○を簡単に説明して」
- 特定の用語や概念についての質問全般

---

## 説明の4ルール（常に適用）

1. **レベルを極限まで落とす**: 専門用語を避け、中学生でも理解できる日常的な言葉で説明
2. **数式よりロジック**: 式より先に「なぜそうなるか」をイメージで説明
3. **算数を使わない解法**: 試験で出たときの「裏技・ハック」を提案
4. **3分ルール**: 説明が長くなりそうなら「まずはこれだけ覚えればOK」に絞る

---

## 解説フォーマット

### 単一用語の解説

```
📖 [用語名]
━━━━━━━━━━━━━━━━━━━━━

🔰 ひとことで言うと:
  [1行で超簡潔に]

🏠 身近な例え:
  [日常生活の例えで説明]

📝 試験ではこう出る:
  [試験での問われ方と覚え方のコツ]

🎯 これだけ覚えればOK:
  ・[ポイント1]
  ・[ポイント2]

━━━━━━━━━━━━━━━━━━━━━
```

### 2つの概念の比較

```
📖 [概念A] vs [概念B]
━━━━━━━━━━━━━━━━━━━━━

🔰 ひとことで言うと:
  [A] = [1行で]
  [B] = [1行で]

🏠 身近な例え:
  [A]は○○、[B]は△△みたいなもの

📝 見分け方（試験のコツ）:
  ・「□□」というキーワードが出たら → [A]
  ・「■■」というキーワードが出たら → [B]

| | [A] | [B] |
|---|---|---|
| 目的 | ... | ... |
| 特徴 | ... | ... |
| 覚え方 | ... | ... |

━━━━━━━━━━━━━━━━━━━━━
```

### 計算・手順系の解説

```
📖 [テーマ名]
━━━━━━━━━━━━━━━━━━━━━

🔰 ひとことで言うと:
  [何をする計算/手順なのか]

👆 指折りで解く方法:
  ステップ1: [やること]
  ステップ2: [やること]
  ステップ3: [やること]
  → 答え: [結果]

⚡ 試験の裏技:
  [計算しないで解く方法があれば提案]

━━━━━━━━━━━━━━━━━━━━━
```

## 基本情報技術者試験の分野判定

質問された用語がどの分野に属するか自動判定し、タグを付ける:

| 分野 | キーワード例 |
|------|-------------|
| 基礎理論 | 2進数、論理演算、確率、BNF、オートマトン |
| コンピュータシステム | CPU、メモリ、OS、割り込み、仮想記憶 |
| データベース | SQL、正規化、ER図、トランザクション |
| ネットワーク | TCP/IP、DNS、DHCP、サブネット、ルーティング |
| セキュリティ | 暗号、認証、ファイアウォール、マルウェア、PKI |
| 開発技術 | ウォーターフォール、アジャイル、テスト、UML |
| マネジメント | PMBOK、ITIL、SLA、WBS、工数見積もり |
| ストラテジ | 経営戦略、マーケティング、財務、法務、DX |
| 科目B アルゴリズム | ソート、探索、再帰、スタック、キュー、木 |
| 科目B 擬似言語 | 変数、配列、ループ、条件分岐、関数 |

## 注意事項

- セッション開始手順は不要（軽量スキル）
- ファイル更新は行わない
- 1回の応答で完結する（追加の対話は最小限）
- 出力はスクロールせずに読める量に収める
- 説明が長くなりそうなら「まずはこれだけ」に絞り、「もっと知りたい？」と聞く
- もっと深く学びたい場合は「○○を勉強したい」と言えば deep-learn スキルが起動する旨を案内する
