# 週次・月次レビュー

## 起動時の確認

ユーザーに「週次レビューですか？月次レビューですか？」と確認する。
指定がなければ、月初なら前月の月次、それ以外は週次をデフォルトにする。

## ワークフロー

### 1. ジャーナル集計

対象期間のジャーナルファイル（`journal/YYYY/MM/YYYY-MM-DD.md`）を読み取る。

集計する項目：
- **気分・体調の推移**: 各日の評価（1-5）を表にまとめる
- **学習記録**: 学習した日数、内容の概要
- **やったこと**: 各日の「今日やったこと」を統合リスト化
- **感謝・良かったこと**: 各日の記録を統合リスト化

### 2. できたことリスト

ジャーナルの「今日やったこと」から、期間中の成果を箇条書きで整理する。
小さなことでも含める。INFPの自己批判傾向に対し、**客観的な事実**として提示する。

### 3. キャリアロードマップ進捗

`profile/キャリアロードマップ.md` を読み、現在のPhaseの進捗を確認する。
TODOリストの完了状況を表示する。

### 4. 習慣達成率（月次のみ）

`health/_template_habit.md` または該当月の習慣トラッカーを読み、達成率を計算する。

### 5. 次の期間の目標設定

振り返りを踏まえて、次の週/月の目標を**2〜3個**提案する。
ユーザーと相談して決定し、必要に応じてタスクファイルに記録する。

## 出力フォーマット

```
## [週次/月次] レビュー（YYYY-MM-DD 〜 YYYY-MM-DD）

### 気分・体調の推移
| 日付 | 気分 | 体調 | 睡眠 | エネルギー |
|------|------|------|------|-----------|
| ... | ... | ... | ... | ... |

### できたこと
- ...

### 学習の記録
- 学習した日数: X / Y 日
- 主な内容: ...

### キャリア進捗
- 現在のPhase: ...
- 完了済みTODO: X / Y

### 次の目標（提案）
1. ...
2. ...
3. ...
```

## 注意事項

- ジャーナルが存在しない日はスキップする（責めない）
- データが少なくても無理に分析せず、「記録を増やしていきましょう」と提案する
- 良かった点を先に、改善点は後に伝える
