# 開発活動

## Roast Plus（業務アプリ）

- **開始**: 2025年7月
- **コミット数**: 555超
- **利用者**: 同僚8名が実際に使用中
- **内容**: コーヒー抽出のガイダンスや業務改善に活用
- **担当範囲**: ビジネス分析から実装・保守まで一人で担当
- **技術**: Next.js（FlutterからWebアプリに移行）
- **管理**: GitHub Issuesベースのプロジェクト管理
- **最近の取り組み**: Claude Codeを使ってUI一貫性の改善

### 開発のきっかけ

今働いている場所での業務改善アプリを作ってみようと思い、生まれたのがローストプラス。最初はFlutterで作っていたが、まいカゴをストアにリリースする大変さを知り、手軽にデプロイ可能なNext.jsを採用してWebアプリ化した。

## なぜITなのか

アプリを作ったことで、自分がソフトウェアエンジニアとして社会に貢献したり人の役に立てる可能性を感じた。同僚が自分の作ったアプリを実際に使っているのを見て、問題解決型アプリ開発への情熱を発見した。

また、バイブコーディングの限界や、知識のないエンジニアリングの危険性、自分が求めているものを作れないこと、利用者からの信頼性のなさなども考慮し、**基礎的な学習こそAI時代に必要**だと強く感じた。
