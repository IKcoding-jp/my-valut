# 問題集学習用プロンプト

以下では、**問題集の「わからない」を最速で解決するためのプロンプト体系（テンプレート）**を、目的別に提示します。  
生成AIにそのまま入力でき、G検定のような知識試験でも高い効果があります。

---

## 1. わからない問題に対して使うべきプロンプト（最小セット）

### ①「何が理解できていないかを特定してほしい」

理解のボトルネックを最短で見つけるための質問です。

**プロンプト：**  
「この問題が理解できません。私がつまずいている前提知識や誤解を特定してください。  
問題文：〇〇  
私の理解：△△」

→ AIが"理解不足の構造"を可視化します。

---

### ②「解説のどこをどう読むべきか整理してほしい」

解説が長い、抽象的、飛躍があるときに使う。

**プロンプト：**  
「次の解説の論理構造を、因果関係と重要ポイントに分解して説明してください。  
また、どこが理解の壁になっているかを指摘してください。  
解説：□□□」

→ "意味がつながらない"状態を解消できます。

---

### ③「図示・比喩・例を使って説明してほしい」

専門用語や抽象概念に詰まった時に効果的。

**プロンプト：**  
「この概念を、

1. 初心者向けの平易な言葉
    
2. 図解的な構造説明
    
3. 具体例  
    の3ステップで説明してください。  
    対象：〇〇」
    

→ 抽象→具体の階段を作ってくれる。

---

## 2. 「なぜこの選択肢が正しい（誤り）なのか」理解したいときのプロンプト

### ④「各選択肢が正しい／誤りとなる理由を比較して説明」

**プロンプト：**  
「次の選択肢を、  
・何が正しいのか  
・なぜ正しく（誤って）いるのか  
・どう区別すべきか  
を比較表で説明してください。  
問題：〇〇  
選択肢：A〜D」

→ 似た選択肢の区別ができるようになります。

---

## 3. 自分の誤解ポイントを明確にしたいとき

### ⑤「私の理解のどこがズレているか修正してほしい」

**プロンプト：**  
「この問題について、私の理解のどこが誤解なのか、正しい知識との差分を明確に示してください。  
問題：〇〇  
私の理解：△△」

→ 誤解を起点に修正するので、記憶定着が強くなります。

---

## 4. どうしても理解できない概念があるときのプロンプト

### ⑥「別の視点・別の言い換えで説明してほしい」

**プロンプト：**  
「この説明がまだ理解できません。別の視点・言い換え・例を用いて、3種類の異なる説明をしてください。  
対象：〇〇」

→ 説明バリエーションを出すことで、自分にフィットする理解方式が見つかる。

---

## 5. 問題を解くための思考プロセスを学びたいとき

### ⑦「正解に至る"考え方"を段階的に示してほしい」

**プロンプト：**  
「この問題を解くときの思考プロセス（着眼点・判断基準・除外基準）を段階的に示してください。  
問題：〇〇」

→ 単なる解説ではなく、"次の問題でも応用できる"思考法が得られます。

---

## 6. AIを「家庭教師」として使う高度なプロンプト

### ⑧「理解確認テストで弱点をあぶり出してほしい」

**プロンプト：**  
「この問題が理解できていないようなので、理解に必要な前提知識が身についているか確認するためのミニテストを5問作ってください。回答後に弱点分析もお願いします。」

→ 自分が気づいていない穴も検出できます。

---

## 7. 問題学習における"最強の使い方"まとめ（3ステップ）

1. **自分の理解を書き出す**
    
2. **AIに"誤解・前提不足・論理飛躍"を特定させる**
    
3. **必要に応じて図解・例示・比較・再テストを依頼**
    

これにより、問題集の不明点はほぼ解消できます。

# 理解を深めるためのプロンプト（最も効果が高い）

## 例1：自分の理解の穴を明確にする

```
この概念を自分なりにこう理解しています：[自分の理解を書く]。

この理解のどこが正しくて、どこが不足していますか？

また、より正確に理解するための補足を教えてください。
```

**使い方のポイント：**
- 自分の理解を先に提示することで、AIが具体的なフィードバックを返せる
- 間違いを恐れず、素直に自分の理解を書くことが重要

---

## 例2：プロセスを分解して説明させる

```
この問題を解くための思考プロセスを、初学者にも分かる段階的ステップで説明してください。

最終回答よりも「考え方」を重視して解説してください。
```

**使い方のポイント：**
- 答えだけでなく、思考プロセスを理解することが重要
- 段階的に説明させることで、自分の思考の型を作れる

---

## 例3：自分の文章を採点させる

```
この説明を、試験官の立場で採点するとどうなりますか？

点数・良い点・改善点を具体的に教えてください。
```

**使い方のポイント：**
- 客観的な視点で自分の説明を評価してもらう
- 改善点を具体的に知ることで、次回の説明力を向上できる

---

# 知らない言葉・用語を勉強するプロンプト

## 例14：初学者向けの段階的説明（最適）

```
「[知らない言葉]」を、完全に知らない初学者にも分かるように説明してください。

箇条書きを優先し、視覚的・具体的な例を多用して、以下の形式で：

1. 一言で言うと何か（簡潔な定義＋視覚的な比喩）
2. なぜその名前なのか（語源・由来・連想イメージ）
3. 具体例を3つ以上（実例・コード例・視覚的比喩）
4. 似ている言葉との違い（混同しやすい用語があれば箇条書きで）
5. 実際にどう使うか（実践例・エンジニア転職向け例・コード例）
6. 記憶に残る覚え方（語呂合わせ・視覚的関連付け・日常比較）

【重要】抽象的表現は避け、箇条書きで要点を明確に。初心者向けに調整。
```

**使い方のポイント：**
- 完全に知らない言葉でも、段階的に理解できる
- 具体例と実例で記憶に定着しやすい
- 関連用語との違いを理解することで、誤用を防げる
- 記憶に残る覚え方で、長期記憶に結びつく

---

## 例15：自分の理解レベルを提示して学習

```
「[知らない言葉]」について、私は以下のレベルです：
- 聞いたことはあるが意味は分からない
- なんとなく分かるが説明できない
- 完全に初めて聞く言葉

このレベルに合わせて、以下の内容を教えてください：
1. 初学者向けの分かりやすい定義
2. なぜ重要なのか、どんな場面で使われるか
3. 具体例（できれば私が知っている分野の例）
4. 関連する用語や概念
5. 実際に使えるようになるための練習方法

私の理解レベルに合わせて、段階的に説明してください。
```

**使い方のポイント：**
- 自分の理解レベルを明確にすることで、適切な説明を受けられる
- 既知の分野の例を使うことで、理解が深まる
- 練習方法まで聞くことで、実践的に使えるようになる

---

## 例16：語彙を体系的に学習

```
「[知らない言葉]」と関連する用語を、以下の形式で整理してください：

1. 核心概念：[知らない言葉]の本質的な意味
2. 関連用語：同じカテゴリの用語3〜5個（それぞれ簡潔に説明）
3. 対比用語：似ているが異なる用語2〜3個（違いを明確に）
4. 上位概念：これを含むより大きな概念
5. 下位概念：これに含まれる具体的な概念

各用語について、初心者にも分かる具体例を1つずつ添えてください。
```

**使い方のポイント：**
- 単語を孤立して覚えるのではなく、体系として理解できる
- 関連用語と一緒に覚えることで、記憶が強化される
- 概念の階層を理解することで、応用が効く

---

## 例17：実践的な使い方を学ぶ

```
「[知らない言葉]」について：

1. 定義を簡潔に（1〜2文）
2. 実際に使われる場面を3つ以上（具体的なシナリオで）
3. 正しい使い方の例文を3つ
4. 間違った使い方の例文を2つ（なぜ間違いかも説明）
5. この言葉を使いこなすための練習問題を2問

実務や日常で使えるレベルまで理解できるようにお願いします。
```

**使い方のポイント：**
- 正しい使い方だけでなく、間違った使い方も知ることで誤用を防げる
- 練習問題で理解を確認できる
- 実務で使えるレベルまで学習できる

# 演習の質を上げるプロンプト

## 例4：レベル調整可能な練習問題生成

```
[単元名]について、理解度確認用の練習問題を5問作ってください。

難易度は「基礎」「応用」「試験レベル」の3段階でお願いします。

解説は簡潔ではなく、なぜその答えになるかを詳細に書いてください。
```

**使い方のポイント：**
- 自分のレベルに合わせて問題を生成できる
- 詳細な解説で理解を深められる

---

## 例5：間違い分析特化

```
以下の問題を自分はこう解きました：[自分の解答]。

どこで思考がズレた可能性があるか、原因と改善策をセットで教えてください。
```

**使い方のポイント：**
- 間違いから学ぶことで、同じミスを繰り返さない
- 原因と改善策をセットで聞くことで、次回の対策が明確になる

# 応用力を伸ばすプロンプト

## 例6：パターン学習 → 本質理解へ

```
この問題と同じパターンの問題を3つ作ってください。

さらに、異なるパターンだが同じ本質を問う問題も2つ作ってください。
```

**使い方のポイント：**
- パターン認識力を高める
- 本質を理解することで、応用問題にも対応できるようになる

---

## 例7：実務・実例に結びつける

```
この概念は、現実のどんな場面で使われますか？

具体的な実例を3つ以上挙げ、それぞれ短いストーリーで説明してください。
```

**使い方のポイント：**
- 抽象的な概念を具体例で理解する
- 実務との関連性を理解することで、学習意欲が高まる

# メタ認知（成績が上がる人が必ずやる）

## 例8：自分の弱点の特定

```
今日学んだ内容のうち、自分が見落としやすいポイントや誤解しやすい点をリストアップしてください。

その補強方法も教えてください。
```

**使い方のポイント：**
- 自分の弱点を客観的に把握する
- 補強方法まで聞くことで、具体的な行動に移せる

---

## 例9：学習ログの自動構築

```
いま理解した内容を、他の人に説明できるレベルの文章にまとめてください。

重要語句、因果関係、例、比較、落とし穴を含めて説明してください。
```

**使い方のポイント：**
- 人に説明できるレベルまで理解を深める（ラーニングピラミッド）
- 構造化された説明で、知識を定着させる

---

## 例10：次の学習ステップを提案

```
この単元を完全に理解するために、次に何を学ぶべきですか？

優先順位と理由を添えて、3〜5つ提示してください。
```

**使い方のポイント：**
- 学習の方向性を明確にする
- 優先順位を理解することで、効率的に学習できる

# 試験対策特化プロンプト

## 例11：過去問の弱点抽出

```
以下の間違えた問題の共通点を分析し、弱点カテゴリを特定してください。

改善のための重点学習項目を出してください。

[間違えた問題リスト]
```

**使い方のポイント：**
- 間違いのパターンを分析することで、根本的な弱点を特定
- 重点学習項目を明確にすることで、効率的に改善できる

---

## 例12：試験本番モード

```
[試験名]の本番に近い形式で10問作り、タイマー形式で出題してください。

回答後は、点数・平均解答時間・改善点を分析してください。
```

**使い方のポイント：**
- 本番に近い環境で練習できる
- 時間管理の練習も同時に行える

# AIの使い方を上手にするための"型プロンプト"

成績が伸びる人は、このような「思考ベースのプロンプト」をよく使います。

## 例13：AIに出す前に「自分の考え」を必ず入れる型

```
【自分の考え】

[ここに短く書く]

【AIに求めたいこと】

誤り・不足・改善ポイントを教えてください。

また、より深く理解するための追加説明や例もお願いします。
```

**使い方のポイント：**
- これは最も効果が高いプロンプト型
- 自分の考えを先に提示することで、AIが質の高いフィードバックを返せる
- 構造化されているため、使い回しが容易

---

## まとめ：いいプロンプトとは「思考・目的・検証」がある

成績が上がる人のプロンプトには共通点があります。

### 良いプロンプトの4つの要素

1. **自分の考えを提示する**（AI が質の高いフィードバックを返せる）
2. **プロセス重視で説明を要求する**（答えだけでなく、思考プロセスを理解）
3. **解説を検証し、理解を深めるための追加質問をする**（深い理解を目指す）
4. **今後の学習ステップまで AI に管理させる**（継続的な学習をサポート）

---

## 使い方のコツ

### プロンプトを効果的に使うためのポイント

1. **自分の理解を先に書く**
   - 間違いを恐れず、素直に自分の理解を提示する
   - AIは具体的なフィードバックを返せるようになる

2. **プロセスを重視する**
   - 答えだけでなく、「なぜそうなるか」を理解する
   - 思考の型を作ることで、応用問題にも対応できる

3. **継続的に改善する**
   - フィードバックを受けたら、それを元に再度質問する
   - 理解が深まるまで繰り返す

4. **学習ログを残す**
   - 重要な理解や気づきは記録に残す
   - 後で振り返って復習できるようにする

## カスタマイズ例

### 特定の分野に特化したプロンプト

```
【自分の考え】
[プログラミング/数学/言語学習など、特定分野の理解]

【AIに求めたいこと】
- 誤り・不足・改善ポイント
- より深く理解するための追加説明や例
- 実践的な練習方法
```

### 複合的なプロンプト

```
以下の内容について：
1. 自分の理解を確認
2. 練習問題を生成
3. 弱点を特定
4. 次の学習ステップを提案

[学習内容]
```
