# Obsidian Vault - フォルダ構造ガイド

このObsidianボルトは、学習記録、プロジェクト管理、知識管理を統合したシステムです。

## 📁 フォルダ構造

```
00_Inbox/              # 未整理の入力（LINE、アイデア、タスクなど）
01_Clippings/          # 外部からのクリッピング（未処理）
02_Projects/           # アクティブなプロジェクト
03_Learning/           # 学習記録
04_Knowledge/          # 知識ベース（文献ノート、永続ノート、リソース）
05_Archive/            # アーカイブ（古い/完了したコンテンツ）
```

## 📋 各フォルダの説明

### 00_Inbox/
**用途**: 未整理のアイデア、タスク、外部からの入力（LINEなど）を一時的に保存

**内容**:
- `inbox.md` - 未整理のアイデアやメモ
- `tasks.md` - 未整理のタスク
- `external/LINE/` - LINEから取得したメッセージ

**ワークフロー**: 定期的に整理し、適切な場所に移動

### 01_Clippings/
**用途**: 外部ソース（Kindle、Web記事など）からのクリッピングを保存

**内容**:
- `Kindle Highlight/` - Kindleハイライト

**ワークフロー**: クリッピングを処理し、文献ノートとして`04_Knowledge/literature/`に移動

### 02_Projects/
**用途**: アクティブなプロジェクトとアイデアを管理

**内容**:
- `active/` - 進行中のプロジェクト
- `ideas/` - プロジェクトアイデア、やりたいことリスト

**ワークフロー**: 完了したプロジェクトは`05_Archive/projects/`に移動

### 03_Learning/
**用途**: 学習記録（日次・週次）を管理

**内容**:
- `daily/` - 日次学習記録
- `weekly/` - 週次振り返り
- `templates/` - テンプレートファイル
- `index.md` - 学習記録のインデックス

**ワークフロー**: 古い記録は`05_Archive/learning/`に移動

### 04_Knowledge/
**用途**: 知識ベース（Zettelkasten方式）

**内容**:
- `literature/` - 文献ノート（書籍、記事から得た知識）
- `permanent/` - 永続ノート（自分の言葉で再構成した知識）
- `resources/` - リソース（プロンプト集、技術書リストなど）
- `index.md` - 知識ベースのインデックス

**ワークフロー**: 
1. クリッピング → 文献ノート
2. 文献ノート → 永続ノート（重要な概念を抽出）

### 05_Archive/
**用途**: 古い/完了したコンテンツをアーカイブ

**内容**:
- `learning/` - 古い学習記録
- `projects/` - 完了したプロジェクト
- `clippings/` - 処理済みクリッピング

## 🔄 ワークフロー

### 1. 入力の整理（Inbox → 適切な場所）
- **毎週日曜日**または**20件を超えたら**整理
- プロジェクト関連 → `02_Projects/`
- 知識ベース → `04_Knowledge/`
- タスク → `tasks.md`で管理

### 2. クリッピングの処理
- `01_Clippings/`のクリッピングを読む
- 重要な内容を文献ノートとして`04_Knowledge/literature/`に作成
- 処理済みクリッピングは`05_Archive/clippings/`に移動

### 3. 知識の定着
- 文献ノートから重要な概念を抽出
- 永続ノートとして`04_Knowledge/permanent/`に作成
- 他のノートとリンクして知識ネットワークを構築

### 4. プロジェクトの管理
- アクティブなプロジェクトは`02_Projects/active/`で管理
- 完了したプロジェクトは`05_Archive/projects/`に移動

## 📝 命名規則

- **文献ノート**: `[著者名]-[タイトル].md`
- **永続ノート**: `YYYY-MM-DD-[タイトル].md` または `[タイトル].md`
- **学習記録**: `YYYY-MM-DD.md`（日次）、`YYYY-WXX.md`（週次）
- **プロジェクト**: `[プロジェクト名].md`

## 🔗 リンクの使い方

- `[[ファイル名]]` - 同じフォルダ内のファイル
- `[[フォルダ/ファイル名]]` - 別フォルダのファイル
- `[[../フォルダ/ファイル名]]` - 親フォルダからの相対パス
