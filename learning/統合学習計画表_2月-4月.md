## 学習方針

### 基本原則

- [[学習の心得]]: 栢木先生の5つのポイントを常に意識する
- **「書かない」学習（ごまどう式）**: ノートは一切取らず、読む・解くの回転数を最大化する
- **80%で安定が目標**: 100%を目指すと過学習・時間浪費のリスクあり
- 間違えることが勉強。間違えた問題は必ず参考書で復習
- 頻出分野（セキュリティ、アルゴリズム）を優先的に学習
- **3分ルール**: 3分悩んで分からなければClaudeに即質問

### 科目B攻略：「動画インプット → トレース練習 → 問題演習」の3ステップ

**背景**: テキスト読み込みでは「全部わからない」状態だった。視覚的な入り口に変更することで理解と定着を高める。

```
3ステップ学習フロー：
① 【動画】すーさんの YouTube で「なぜそうなるか」を視覚的に理解
   → プログラムの動作フロー・メモリ管理・データ構造を「図」で把握

② 【テキスト】「出るとこだけ！科目B」でトレース技法を習得
   → 理解した内容を「手で書く・動かす」で定着させる
   → 選択肢「ありえない選択肢テクニック」（第4章）を活用

③ 【問題】IPAサンプル + 過去問道場で実戦演習
   → テキスト参照なしで解く
   → わからない時だけ動画に戻る（ピンポイント復習）

**狙い**:
- 知覚推理（112）の強み活かし：「文字」ではなく「図・映像」から入る
- 言語理解（70）の弱点回避：アルゴリズムを「言語説明」ではなく「動作」で理解する
- 2.5ヶ月の余裕活用：焦らず、丁寧に「理解→定着→実戦」を進める
```

### 科目Bの解き方（4段階方式）

1. **まずコードを読んで答えを出してみる**（表は書かない）
2. 読むだけでわからなければ → **聞かれてる変数だけメモしてトレース**
3. それでもダメなら → **選択肢から逆算**（第4章のテクニック）
4. 3分悩んでダメなら → **★マークで飛ばす**

### 基礎理論の段階的学習方針

> **方針: 基礎理論は「捨てない」。ただし優先順位をつけて、取れる問題から確実に広げる。**
> **学習スタイル: 小学校の算数レベルから丁寧に。Claudeが一からサポートする。**

#### 段階1: 🔴 最優先（手順を覚えれば解ける）
- **2進数・基数変換**: 10進数↔2進数↔16進数の変換。指折りでできる
- **論理演算**: AND/OR/NOT/XOR。「1＝ON、0＝OFF」のスイッチで考える
- **集合**: ベン図で視覚的に理解

#### 段階2: 🟡 できれば（公式＋パターンで対応）
- **確率の基礎**: 「全部の場合の数」÷「当たりの数」だけ
- **待ち行列の基本公式**: 利用率＝到着率÷サービス率
- **オートマトン**: 迷路をたどるパズルとして解く

#### 段階3: 🔵 余裕があれば
- **浮動小数点**: 仕組みだけ理解し、典型問題を1〜2問解く
- **情報理論**: 頻出パターンのみ

### 隙間時間の黄金ルール（科目A専用：昼休み・夜・移動中）

- **短期（15分・移動）**: 過去問道場で5問解く。解説をその場で熟読
- **中期（30〜40分・昼休）**: 過去問道場の「分野別」で重点演習
- **共通**: スマホで過去問道場をブックマークして常に開く
- **わからない問題**: スクショしてClaudeに質問 → 即解決

### AIの活用ルール

#### 3分ルール
- **3分悩んでも分からない** → すぐClaudeに質問
- 時間を無駄にしない。AIを「即席の家庭教師」として使う

#### 質問の仕方
1. 問題文をスクショまたはコピペ
2. 「この問題がわかりません」と伝える
3. **中学生でも分かる説明**を求める

#### 外出先では
- Claude.aiを使用（スマホブラウザ）
- 帰宅後、つまずきポイントをClaude Codeに共有してフォローアップ

## 成功のための原則

### 1. 認知特性の最大活用

- **知覚推理（PRI = 112）を優先**:
  - 視覚的な学習（動画 → テキスト）
  - 図形・パターン認識で問題を解く
  - ありえない選択肢を「パターンで消去」

- **言語理解（VCI = 70）の課題を避ける**:
  - 長い説明文は図に変換
  - トレース表を手で大きく書く
  - 「完全理解」より「パターン習得」

### 2. 試験を最優先

- 4月19日までは基本情報試験が最優先
- プログラミングは「基礎固め」のレベルに留める
- 本格開発は試験後（4月下旬～）

### 3. メンタルヘルス重視

- 「焦り」は理解を浅くする → 焦らないペース設定
- 「無理しない判断」を許容する文化
- 疲労時は「最低限パターン」で継続

### 4. 過学習防止

- 同じ問題の繰り返しではなく、ランダム出題で本当の理解力を試す
- 得意分野に逃げず、弱点を戦略的に補強
- 「80%安定」を目標とし、100%を目指さない

## 非常時の対応

### 予期しない遅延（体調不良など）

1. **その日の学習を無理せず休む**
2. **翌日 AI（Claude Code）に報告**
3. **スケジュール調整を相談**
4. **遅延分をどう回収するか検討**

### メンタル低下時

1. **`/mental-coach` スキルを活用**
2. **無理な学習は避ける**
3. **「核心の言葉」を思い出す**: 「つらいけど、しんどいけど、私は幸せになりたい」

### 試験直前期（4月中旬）での判断
