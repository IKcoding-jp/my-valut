## 問題
「社員」表から、所属部門が'営業'である社員の氏名を取得するSQL文として正しいものはどれか。

| 社員番号 | 氏名 | 所属部門 |
|---------|------|---------|
| 001 | 田中太郎 | 営業 |
| 002 | 鈴木花子 | 開発 |
| 003 | 佐藤次郎 | 営業 |

## 選択肢
- ア: SELECT 氏名 FROM 社員 WHERE 所属部門 = '営業'
- イ: SELECT 所属部門 FROM 社員 WHERE 氏名 = '営業'
- ウ: SELECT 氏名 FROM 社員 WHERE 所属部門 LIKE '営業%'
- エ: SELECT * FROM 社員 ORDER BY 所属部門 = '営業'

## 正解
ア

## 解説

### 正解の理由
SQL文の基本構造は「SELECT（何を取得するか）FROM（どの表から）WHERE（どの条件で）」である。
- 取得したいのは「氏名」→ SELECT 氏名
- 対象の表は「社員」→ FROM 社員
- 条件は「所属部門が営業」→ WHERE 所属部門 = '営業'

この3つを組み合わせると、選択肢アが正しい。

### 各選択肢の解説
- ア: 正しいSQL文である。SELECTで氏名を指定し、WHEREで所属部門が'営業'という条件を指定している。
- イ: SELECTとWHEREの指定内容が逆になっている。これでは「氏名が'営業'の行の所属部門」を取得しようとしており、意味が通らない。
- ウ: LIKE '営業%' は「営業で始まる文字列」を検索するパターンマッチであり、完全一致ではない。今回は完全一致で十分であり、最も適切とは言えない。
- エ: ORDER BYは並び替えの指定であり、条件による絞り込み（WHERE）ではない。全社員が取得されてしまう。
