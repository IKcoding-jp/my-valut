# 過去問道場データ

過去問道場からエクスポートした成績レポートCSVを保管するフォルダ。

## 運用方針：1回分（60問）ごとに取り込み

全データを毎回上書きするのではなく、**1回の演習（60問）ごとにCSVを保存**する。

**理由**: 全データだと得意分野ばかり解いて正答率が上がったように見えるバイアスが隠れる。
1回ごとなら「何をやったか」「苦手を避けてないか」「回ごとの成長」が正直に見える。

## 使い方

1. 過去問道場で1回分（60問）を解き終える
2. 成績ページから「CSVエクスポート」でダウンロード
3. このフォルダに保存（ファイル名はデフォルトのままでOK）
4. Claude Codeに「CSVを取り込んで」と伝えれば `進捗ログ.md` に反映

## questions/ との役割分担

過去問道場の問題は `questions/` に1問ずつファイル化**しない**。

| データソース | 記録先 | 粒度 | 目的 |
|---|---|---|---|
| **過去問道場** | `過去問道場データ/*.csv` | CSVに全問の正誤 | 量の記録（正答率推移・分野バランス） |
| **/study スキル演習** | `questions/` 個別ファイル | 自信度・解答時間・メタ認知 | 質の記録（本当に理解したかの確認） |

**理由**: 過去問道場にはまぐれ正解も含まれる。正解＝理解とは限らないので、
`questions/` には /study で深掘りして自信度を確認した問題だけを記録する。

**間違えた問題の復習フロー**:
1. 過去問道場で模試 → CSVで記録
2. 間違えた問題のうち気になるものを /study で復習
3. /study で扱った問題だけ questions/ に詳細記録

## ファイル仕様

- **エンコーディング**: cp932（Shift_JIS系）
- **ファイル名例**: `report202602011830.csv`（日時が自動付与される）
- **カラム**: No., 正誤, 系統, 中分類, 小分類, 出典, 学習日 等
