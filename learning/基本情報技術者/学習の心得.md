# 学習の心得

## 栢木先生の5つのポイント

『かやのき先生の基本情報技術者教室』より。

1. **勉強時間は作るもの** — 通勤中、昼休み、トイレの中。毎日やれば日課になる
2. **最初から完璧を目指さない** — 分からなくても次へ進む。後から「こういうことか」と繋がる
3. **理解できない問題が出たら喜ぶ** — 「新しい知識が身につく」とプラス思考で
4. **ネットの掲示板を活用** — 同じ目標の人と刺激し合う（Claude Codeも活用）
5. **理解できない問題は共有する** — 一人で抱え込まない

## 基礎理論の段階的学習

> 方針: 基礎理論は「捨てない」。優先順位をつけて、取れる問題から確実に広げる。

### 段階1: 🔴 最優先（手順を覚えれば解ける）
- **2進数・基数変換**: 10進数↔2進数↔16進数。指折りでできる
- **論理演算**: AND/OR/NOT/XOR。「1＝ON、0＝OFF」のスイッチ
- **集合**: ベン図で視覚的に理解

### 段階2: 🟡 できれば（公式＋パターン）
- **確率の基礎**: 「全部の場合の数」÷「当たりの数」だけ
- **待ち行列の基本公式**: 利用率＝到着率÷サービス率
- **オートマトン**: 迷路をたどるパズルとして解く

### 段階3: 🔵 余裕があれば
- **浮動小数点**: 仕組みだけ理解し、典型問題を1〜2問
- **情報理論**: 頻出パターンのみ

## 1日のスケジュール（毎日共通）

| 時間 | 内容 |
|------|------|
| 5:00 | 起床 |
| 5:30 - 7:30 | **朝4ポモドーロ（科目B → 科目A）** |
| 7:30〜 | 身支度・朝食・出発 |
| 通勤・昼休み | 隙間時間（Anki・過去問道場 数問） |
| 帰宅後 | **自由時間（勉強しない）** |

- **1日の勉強時間**: 約2時間（4ポモドーロ）+ 隙間時間
- 調子が良くても**4ポモドーロで止める**（頑張りすぎ→離脱を防ぐ）
- 娯楽の時間は削らない。勉強以外の時間は休む

### 3つの方針

1. **平日も休日も同じルーティン**を回す
2. **4ポモドーロ以上やらない**（頑張りすぎ→離脱の経験済み）
3. **楽しくない日も5:30に座ることだけ守る**

## 成功のための4原則

1. **認知特性の最大活用**: 知覚推理（PRI=112）を優先。視覚的学習、パターン認識で問題を解く
2. **試験を最優先**: 4月19日まで基本情報が最優先。プログラミングは基礎固めレベルに留める
3. **メンタルヘルス重視**: 「焦り」は理解を浅くする。疲労時は最低限パターンで継続
4. **過学習防止**: ランダム出題で本当の理解力を試す。「80%安定」が目標、100%は目指さない

## 遅延・非常時の対応

### 遅延が発生した場合

| 遅延幅 | 対応 |
|--------|------|
| 1-3日 | その週の残り日数で回収。無理なら次週に繰り越し |
| 1週間以上 | Phase内の優先度を再調整 |
| 1フェーズ全体 | Claude Codeと相談の上、試験日変更も検討 |

### メンタル低下時

1. `/mental-coach` スキルを活用
2. 無理な学習は避ける
3. 核心の言葉を思い出す: 「つらいけど、しんどいけど、私は幸せになりたい」

### 試験直前期（4月中旬）

- 焦りが見えたら → 最低限パターンに切り替え
- 復習が足りない → Phase 4の調整で対応
- **新しい内容の追加は避ける**
